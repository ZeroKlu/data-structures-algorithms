Section 11 - Problems

Graph Algorithms - Problem Set

Core graph patterns: BFS & DFS, connected components, topological sorting, shortest paths, and union-find. Each problem is designed to exercise a specific pattern you'll see repeatedly in interviews and real-world code.

Problem 1 - Shortest Path in an Unweighted Graph

You are given an undirected, unweighted graph with n vertices labeled 0..n-1 and a list of edges. You are also given two vertices src and dst. Using Breadth-First Search (BFS), compute the length of the shortest path from src to dst. If there is no path, return -1.

Input Format

  • An integer n, the number of vertices.
  • An integer m, the number of edges.
  • m lines, each with two integers u v describing an undirected edge.
  • One line with two integers src dst.

Output Format

A single integer: the minimum number of edges on any path from src to dst, or -1 if no such path exists.

Constraints

  • 1 ≤ n ≤ 2 * 105
  • 0 ≤ m ≤ 2 * 105
  • 0 ≤ u, v, src, dst < n
  • The graph may be disconnected.

Example

Input
5 4
0 1
1 2
1 3
3 4
0 4

Output
3

Explanation
One shortest path is 0 → 1 → 3 → 4 (3 edges).

Problem 2 - Connected Components in an Undirected Graph

You are given an undirected graph with n vertices labeled 0..n-1 and a list of edges. Using DFS or BFS, compute the number of connected components in the graph.

Input Format

  • An integer n, the number of vertices.
  • An integer m, the number of edges.
  • m lines, each with two integers u v describing an undirected edge.

Output Format

A single integer: the number of connected components in the graph.

Constraints

  • 1 ≤ n ≤ 2 * 105
  • 0 ≤ m ≤ 2 * 105
  • 0 ≤ u, v < n

Example

Input
6 3
0 1
2 3
4 5

Output
3

Explanation
The components are {0,1}, {2,3}, and {4,5}.

Problem 3 - Topological Ordering of a DAG

You are given a directed graph with n vertices labeled 0..n-1 and m directed edges. The graph is guaranteed to be a DAG (Directed Acyclic Graph). Your task is to output a valid topological ordering of all vertices.

Input Format

  • An integer n, the number of vertices.
  • An integer m, the number of edges.
  • m lines, each with two integers u v describing a directed edge u → v.

Output Format

A permutation of 0..n-1 in a single line, representing a valid topological order (if multiple orders are valid, output any one of them).

Constraints

  • 1 ≤ n ≤ 2 * 105
  • 0 ≤ m ≤ 2 * 105
  • 0 ≤ u, v < n
  • The graph is acyclic.

Example

Input
4 3
0 1
0 2
1 3

Output
0 1 2 3

Explanation
Valid topological orders include [0,1,2,3], [0,2,1,3]; 0 must come before 1 and 2, and 1 before 3.

Problem 4 - Single-Source Shortest Path with Non-Negative Weights

You are given a directed weighted graph with n vertices labeled 0..n-1 and m edges. Each edge u → v has a non-negative weight w. Given a source vertex src, compute the shortest distance from src to every vertex using Dijkstra's algorithm.

Input Format

  • An integer n, the number of vertices.
  • An integer m, the number of edges.
  • m lines, each with three integers u v w describing a directed edge.
  • One integer src, the source vertex.

Output Format

A single line with n space-separated integers, where the i-th integer is the shortest distance from src to i, or -1 if i is unreachable.

Constraints

  • 1 ≤ n ≤ 2 * 105
  • 0 ≤ m ≤ 2 * 105
  • 0 ≤ u, v, src < n
  • 0 ≤ w ≤ 109

Example

Input
5 6
0 1 2
0 2 5
1 2 1
1 3 2
2 3 1
3 4 3
0

Output
0 2 3 4 7

Explanation
Example shortest paths:
0→1 (2), 0→1→2 (3), 0→1→3 (4), 0→1→3→4 (7).

Problem 5 - Dynamic Connectivity with Union-Find

You are given n elements labeled 0..n-1. Initially, each element is in its own set. Then you receive q operations of two types:

  • union a b: merge the sets containing a and b.
  • connected a b: query whether a and b are in the same set.

Implement this using a Disjoint Set Union (Union-Find) data structure with path compression and union-by-rank/size.

Input Format

  • An integer n, the number of elements.
  • An integer q, the number of operations.
  • q lines, each in one of the forms:
    • union a b
    • connected a b

Output Format

For every connected query, print "YES" if the two elements are in the same set, otherwise print "NO", each on its own line.

Constraints

  • 1 ≤ n, q ≤ 2 * 105
  • 0 ≤ a, b < n

Example

Input
5 6
union 0 1
union 1 2
connected 0 2
connected 0 3
union 3 4
connected 2 4

Output
YES
NO
NO

Explanation
After unions, we have sets {0,1,2} and {3,4}.
Queries:
- 0 and 2 → YES
- 0 and 3 → NO
- 2 and 4 → NO

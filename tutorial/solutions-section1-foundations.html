Section 1 - Solutions

Foundations - Big-O & Complexity (Solutions)

Step-by-step answers for the Section 1 problem set, focusing on how to turn informal descriptions and small code snippets into clear time and space complexity.

Problem 1 - Dominant Term

The algorithm on input size n does:

  • 5 steps of setup work (constant).
  • A loop that runs 3n times.
  • A nested loop: outer runs n, inner runs 2 times.

Solution

Write the total work as a function of n:

  • Setup: 5 steps.
  • Linear loop: 3n steps.
  • Nested loop: outer n, inner 2 → 2n steps.

Total steps: 5 + 3n + 2n = 5 + 5n.

In Big-O, we:

  • Drop additive constants: 5 + 5n → 5n.
  • Drop constant factors: 5n → n.

Final running time: O(n).

Problem 2 - Time Complexity from Code

Given the pseudocode:

int count_pairs(int arr[], int n) {
    int count = 0;
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            if (arr[i] + arr[j] == 0) {
                count++;
            }
        }
    }
    return count;
}

We want time and space complexity in terms of n.

Solution

The outer loop runs for i = 0 to n - 1, so it runs n times.

For each fixed i, the inner loop runs: j = i + 1, i + 2, ..., n - 1 → about n - i - 1 iterations.

Total number of iterations of the inner body is:

  • When i = 0: n - 1 times.
  • When i = 1: n - 2 times.
  • ...
  • When i = n - 2: 1 time.
  • When i = n - 1: 0 times.

That sum is: (n - 1) + (n - 2) + ... + 1 = n(n - 1)/2, which is O(n²).

So:
Time complexity: O(n²).

Space complexity: the function only uses a few scalar variables (count, i, j) in addition to the input array. There is no extra space that grows with n.
Extra space: O(1).

Problem 3 - Comparing Growth Rates

For large n, we want to rank these from slowest-growing to fastest-growing (all logs base 2):

  1. n log n
  2. 2ⁿ
  3. n²
  4. log n
  5. n³
  6. n

Solution

General rule-of-thumb:

  • Logarithms grow slower than polynomials.
  • Among polynomials, higher powers grow faster (e.g., n³ > n²).
  • Exponentials like 2ⁿ eventually dominate any polynomial.

Applying that, from slowest to fastest:

  1. log n
  2. n
  3. n log n
  4. n²
  5. n³
  6. 2ⁿ

For Big-O comparisons, 2ⁿ is the "explodes fastest" term.

Problem 4 - Space Complexity with an Extra Array

Function: 

function buildPrefix(arr) {
    const n = arr.length;
    const prefix = new Array(n);
    prefix[0] = arr[0];
    for (let i = 1; i < n; i++) {
        prefix[i] = prefix[i - 1] + arr[i];
    }
    return prefix;
}

Solution

The loop runs from i = 1 up to n - 1, so it executes n - 1 iterations. Each iteration does a constant amount of work: one addition and one assignment.

Time complexity: O(n).

For space, we create a new array prefix of length n. That is extra memory proportional to the input size.

Extra space complexity: O(n).

Problem 5 - Nested Loops with Different Bounds

Given the code:

def foo(n: int) -> None:
    i = 1
    while i <= n:
        j = 1
        while j <= i:
            print(i, j)
            j *= 2
        i *= 2

Solution

Outer loop (variable i):

  • i starts at 1 and doubles each time: 1, 2, 4, 8, ..., up to at most n.
  • Number of iterations ≈ the number of times we can double before exceeding n, which is about log₂ n.

Inner loop (variable j):

  • For a fixed i, we set j = 1, then repeatedly do j *= 2 until j > i.
  • Values of j: 1, 2, 4, ..., up to at most i.
  • Number of inner iterations is about log₂ i.

Total work:

We run the inner loop once for each value of i in {1, 2, 4, ..., up to n}. There are about log₂ n such values, and for each one we do about log₂ i inner iterations.

Roughly, the total number of print operations is:

  • ∑ (over i = 1, 2, 4, ..., n) of log₂ i.
  • The largest term in this sum is log₂ n, and there are about log₂ n terms → the sum is on the order of (log₂ n) × (log₂ n).

Therefore, the running time is: O((log n)²).

Extra space: we only use i, j, and a few constant-sized values. There is no extra memory proportional to n, so:
Extra space: O(1).
